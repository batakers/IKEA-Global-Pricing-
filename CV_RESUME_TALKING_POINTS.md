*I built an end-to-end data platform for IKEA pricing analysis. Here's what made it interesting:*

**Problem:** IKEA operates in 46 countries with different pricing strategies. Understanding global pricing patterns requires clean, normalized data and accessible analysis tools.

**Solution:** I created a three-layer architecture:
1. **Data Layer:** ETL pipeline handling 366K+ product records with currency conversion, GDP normalization, and quality validation
2. **Service Layer:** FastAPI REST API with 15 endpoints enabling programmatic access to rankings, benchmarks, and statistics
3. **Presentation Layer:** Streamlit dashboard with interactive visualizations for market comparison and segment analysis

**Key Technical Decisions:**
- Used K-Means clustering for market segmentation (found 4 distinct segments)
- Engineered price and affordability indices for relative market positioning
- Implemented Pydantic for schema validation enforcing business rules
- Deployed with Docker for reproducibility and Streamlit Cloud for accessibility

**Impact:** Stakeholders can now explore 41-country pricing data through a live dashboard, and developers can query the API for custom analyses. 100% test coverage ensures data integrity.
